The Control Layer defines how governance authority is enforced at runtime.

It answers one core question:

Who is allowed to decide, when, and under what conditions?

This layer does NOT define:
- What the system is
- How execution is structured
- Business workflows or application logic

It defines:
- When execution is allowed to proceed
- When execution MUST pause
- When human intervention is REQUIRED
- When execution MUST be blocked or escalated

Without the Control Layer, governance cannot be enforced.
