1. Strategy & Intent Formation
| Decision Step      | Governance Control Point                               |
| ------------------ | ------------------------------------------------------ |
| Intent Declaration | Strategic intent must be explicitly declared and owned |
| Context Validation | Alignment with governance charter and scope            |
| Policy Evaluation  | Strategy consistency with non-overridable principles   |
| Risk Assessment    | Long-term systemic and reputational impact             |
| Authorization      | Governance body approval required                      |
| Execution          | Strategy communicated as non-executable directive      |
| Recovery           | Strategy rollback or re-baselining                     |

2. Resource Allocation
| Decision Step      | Governance Control Point                   |
| ------------------ | ------------------------------------------ |
| Intent Declaration | Allocation intent with quantified scope    |
| Context Validation | Budget, capacity, and authority validation |
| Policy Evaluation  | Compliance with allocation policies        |
| Risk Assessment    | Resource exhaustion and dependency risk    |
| Authorization      | Budget owner / kernel gate approval        |
| Execution          | Allocation tracked and logged              |
| Recovery           | Reallocation or freeze                     |

3. Product / Service Development

| ------------------ | -------------------------------------- |
| Intent Declaration | Product intent documented              |
| Context Validation | Market, capability, and governance fit |

| Risk Assessment    | Quality, liability, and lifecycle risk |
| Authorization      | Development gate approval              |
| Execution          | Controlled build and validation        |
| Recovery           | Rollback, deprecation, or redesign     |
